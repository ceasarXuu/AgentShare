# 探索研究型报告模板

## Overview
按固定结构输出《商业机会探索研究报告》，强调“为什么现在值得继续投入注意力”。

## Workflow
1. 选择一个最值得关注的机会并写一句话结论
2. 汇总事实驱动的“为什么是现在”证据
3. 描述机会背景、用户画像与真实行为证据
4. 分析现有方案结构与由此暴露的机会空间
5. 论证商业化逻辑与价格锚点
6. 说明为何是“小而美”的工具机会并给出结论

## Output Format
使用以下Markdown结构，保持标题与顺序不变：

# 《商业机会探索研究报告》

## 0. Executive Overview（执行概览）
### 研究结论（一句话）
- [一句话结论]

### 为什么是现在（事实驱动）
- [事实证据1]
- [事实证据2]
- [事实证据3]

### 机会类型判定
- [ ] 系统能力不足型
- [ ] 过度工具化型
- [ ] 定价 / 价值失衡型
- [ ] 行为外包型

## 1. 机会的产生背景（Opportunity Genesis）
### 1.1 探索触发点（Exploration Trigger）
- [信号来源与触发说明]

### 1.2 为什么选择这一方向深入研究
- [用户与行为边界]
- [动作→结果闭环]
- [单点工具优势]

## 2. 用户是谁，以及问题发生在什么情境下
### 2.1 核心用户画像（Contextual Persona）
- [用户角色]
- [典型触发场景]
- [具体目标]

### 2.2 用户真实行为证据
- [原话/行为证据1]
- [原话/行为证据2]

## 3. 当前解决方案的结构性特征
### 3.1 市场现有方案的共性结构
- [功能组织方式]
- [使用路径复杂度]
- [定价模型]

### 3.2 由结构自然暴露的机会空间
- [围绕核心动作重新设计的价值空间]

## 4. 商业化逻辑分析（正向）
### 4.1 用户为何愿意为此付费
- [付费/时间/风险成本证据]

### 4.2 可行的商业模式路径
- [模式1+理由]
- [模式2+理由]

### 4.3 价格锚点的现实基础
- [市场定价/心理价位/替代成本]

## 5. 为什么这是一个“小而美”的工具机会
- [功能收敛性]
- [非平台化理由]
- [维护成本可控]

## 6. 总结：为何值得进入下一阶段
- [进入MVP验证/定向研究/持续观察]

## Guardrails
- 2.2节只呈现证据，不做总结性语言
- 不点名批评单一产品
- 结论必须有事实证据支撑
