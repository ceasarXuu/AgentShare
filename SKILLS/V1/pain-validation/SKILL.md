# 痛点验证与信号识别

## Overview
通过三角验证与信号识别将“可能的痛点”收敛为“可行动的痛点”。

## Workflow
1. 为每个痛点收集至少3个独立来源
2. 评估每个来源的信号强度与噪声特征
3. 检查时效性并标注版本/时间范围
4. 判断付费意愿或替代行为
5. 输出已验证痛点与证据链

## 三角验证规则
- 至少3个独立来源支撑同一结论
- 来源类型尽量跨层级（用户行为 + 数据平台 + 论坛/社区）

## 信号 vs 噪声
- 信号特征: 多用户重复、持续存在、愿意付费或自发解决、有清晰上下文
- 噪声特征: 个例、模糊抱怨、极端案例、已有完美免费方案

## 数据时效性
- 仅使用近12个月数据
- 旧数据必须标注版本差异与环境变化

## Output
- 已验证痛点清单: 痛点/证据来源/时间范围/付费意愿迹象
- 待验证痛点与补充动作

## Guardrails
- 无交叉验证的结论只做假设
- 单一来源不进入优先级评估
