# 风险评估

## Overview
系统化识别关键风险并量化其影响与概率。

## Workflow
1. 按四维度列出风险清单
2. 为每项风险评分影响与概率
3. 放入风险-影响矩阵并分类
4. 设计缓解策略与触发条件
5. 输出风险报告与决策建议

## 四维风险
- 技术风险: API可行性/版本兼容/技术验证
- 市场风险: 痛点未验证/竞争红海/需求萎缩
- 监管风险: App Store/隐私/GDPR
- 执行风险: 团队能力/资金runway/时间估算

## 风险-影响矩阵
- 接受区/注意区/严重区/灾难区
- 灾难区必须规避

## Output
- 风险清单与评分
- 风险矩阵与缓解策略
- Go/No-go风险提示

## Guardrails
- 高影响高概率优先处理
- 不能解决的风险必须提前退出
