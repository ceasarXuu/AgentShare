# 决策与MVP定义

## Overview
将调研结果转为可执行的产品决策与最小可行计划。

## Workflow
1. 建立多维评分表并为候选方向打分
2. 结合场景规划确定Go/No-go阈值
3. 定义MVP范围与核心功能
4. 设定成功指标与时间窗口
5. 输出执行路线与里程碑

## 多维评分
- 至少5个维度: 痛点强度/市场潜力/竞争强度/技术可行性/单位经济
- 保持权重一致并记录理由

## MVP定义
- 仅保留1个核心功能
- 3个月内可交付
- 成功指标清晰可量化

## Output
- 决策评分表与结论
- MVP需求清单与成功指标
- 12周执行路线图

## Guardrails
- 不以“愿景”替代可执行路径
- 评分与决策需可复盘
