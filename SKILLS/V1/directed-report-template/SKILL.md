# 命题研究型报告模板

## Overview
按固定结构输出《商业机会命题研究报告》，强调系统性拆解与可行性判断。

## Workflow
1. 明确研究问题与边界
2. 汇总用户需求与行为证据
3. 分析市场与竞品结构
4. 论证商业化与可行性
5. 评估风险与不确定性
6. 给出结论与下一步建议

## Output Format
使用以下Markdown结构，保持标题与顺序不变：

# 《商业机会命题研究报告》

## 0. Executive Summary（执行摘要）
### 研究问题
- [研究命题]

### 核心结论
- [可行/条件可行/不具备商业价值]

### 关键依据
- [市场与需求层面]
- [用户行为与付费意愿]
- [商业模型与实现成本]

## 1. 研究背景与问题定义
### 1.1 背景与研究动因
- [行业/平台/技术背景]
- [研究价值]
- [讨论模糊点]

### 1.2 明确研究问题与边界
- [研究对象]
- [不包含内容]
- [研究假设]

## 2. 用户与需求分析
### 2.1 目标用户与使用场景
- [用户角色]
- [触发场景]
- [目标]

### 2.2 用户行为与需求证据
- [搜索行为]
- [使用频率]
- [讨论与反馈]

## 3. 市场与竞品结构分析
### 3.1 现有解决方案概览
- [产品类型]
- [功能分布]
- [定价模型]

### 3.2 结构性问题与差异点
- [未被满足的需求]
- [过度设计或错配]

## 4. 商业化与可行性分析
### 4.1 商业模式拆解
- [潜在收费方式]
- [收费逻辑与用户心理]

### 4.2 成本与实现复杂度评估
- [技术复杂度]
- [长期维护成本]
- [个人开发可行性]

## 5. 风险与不确定性分析（允许）
- [平台政策风险]
- [需求可持续性]
- [竞争与替代风险]

## 6. 结论与建议
- [是否值得投入]
- [推荐下一步动作]
- [MVP/深度验证/放弃或延后]

## 7. 附录：证据与数据来源
- [原始数据链接]
- [用户原话]
- [截图或表格]

## Guardrails
- 研究问题与边界必须先定义再展开
- 结论必须与证据一致
