# Responsibility & Risk Assessment - 责任与风险评估

## 技能概述

专门评估产品可能带来的各类责任和风险,包括人身安全、财产损失、法律责任、隐私泄露、声誉风险等。这是"好生意评估"七维度模型中的第1维度,也是**最重要的维度**(一票否决)。

## 核心原则

1. **风险优先**: 先看风险,再看收益
2. **最坏情况**: 评估最坏情况能否承受
3. **保守判断**: 有疑虑就不做
4. **不可转嫁**: 免责声明保护力度有限
5. **生命第一**: 涉及安全的产品极度谨慎

## 五类风险评估

### 类型1: 人身安全风险

**定义**: 产品失败或故障可能导致用户人身伤害或生命危险

**高风险示例**:
- 医疗诊断和建议
- 安全监控和报警
- 驾驶辅助和导航
- 运动健康监测(关键场景)
- 老人/儿童安全监护

**评估问题**:
```
□ 产品失败会导致用户受伤吗?
□ 产品给出错误建议会有安全后果吗?
□ 产品承诺了可靠性保障吗?
□ 用户会依赖这个产品保障安全吗?
□ 涉及医疗、驾驶、监护等敏感场景吗?
```

**风险等级判断**:
- **高风险**: 任何一项为"是"
- **中风险**: 间接影响安全(如健康数据记录错误)
- **低风险**: 极小概率影响安全
- **无风险**: 完全不涉及安全

**案例**:
- ❌ 高风险: 老人跌倒检测+自动呼救
- ⚠️ 中风险: 运动数据记录(非关键)
- ✅ 低风险: 健康知识普及(明确非医疗建议)

---

### 类型2: 财产损失风险

**定义**: 产品失败或错误可能导致用户重大财产损失

**高风险示例**:
- 金融投资建议
- 保险理赔处理
- 法律文书生成
- 税务申报工具
- 智能家居控制(涉及财产安全)

**评估问题**:
```
□ 产品给出财务建议吗?
□ 涉及用户的资金操作吗?
□ 涉及法律文书或合同吗?
□ 失败可能导致用户财产损失吗?
□ 损失金额可能超过¥10万吗?
```

**风险等级判断**:
- **高风险**: 可能导致>¥10万损失
- **中风险**: 可能导致¥1-10万损失
- **低风险**: 可能导致<¥1万损失
- **无风险**: 不涉及用户财产

**案例**:
- ❌ 高风险: AI投资顾问(给出买卖建议)
- ⚠️ 中风险: 合同模板生成
- ✅ 低风险: 投资知识教育(不给具体建议)

---

### 类型3: 法律责任风险

**定义**: 产品运营可能触犯法律,导致严重法律后果

**高风险领域**:
- 隐私数据泄露(GDPR, 个人信息保护法)
- 儿童相关(儿童保护法)
- 金融监管(反洗钱、无牌经营)
- 内容审核(违法违规内容)
- 医疗器械(需要资质)

**评估问题**:
```
□ 需要特殊资质/牌照吗?
□ 涉及敏感个人数据(身份证、位置、生物特征)吗?
□ 涉及儿童用户吗?
□ 涉及金融交易吗?
□ 可能出现违法违规内容吗?
□ 需要承担平台责任吗?
```

**风险等级判断**:
- **高风险**: 任何一项为"是"且无明确合规方案
- **中风险**: 有一项为"是"但有合规路径
- **低风险**: 涉及但风险可控
- **无风险**: 完全不涉及

**案例**:
- ❌ 高风险: 儿童社交平台(内容审核+隐私保护)
- ⚠️ 中风险: 企业SaaS(企业数据,有标准合规方案)
- ✅ 低风险: 单机工具(不涉及用户数据上传)

---

### 类型4: 隐私泄露风险

**定义**: 用户隐私数据可能被泄露、滥用或被攻击

**敏感数据分类**:
- **极敏感**: 身份证号、生物特征、医疗记录、定位信息
- **敏感**: 手机号、邮箱、通讯录、聊天记录
- **一般**: 昵称、头像、公开信息

**评估问题**:
```
□ 收集哪些类型的用户数据?
□ 数据存储在哪里?
□ 是否有完善的安全措施?
□ 是否有第三方数据处理?
□ 数据泄露的最坏后果是什么?
```

**风险等级判断**:
- **高风险**: 收集极敏感数据,or 安全措施不足
- **中风险**: 收集敏感数据,有基本安全措施
- **低风险**: 只收集一般数据
- **无风险**: 不收集用户数据(纯本地)

**案例**:
- ❌ 高风险: 人脸识别+云端存储(小团队无安全能力)
- ⚠️ 中风险: 账号系统(手机号+密码,标准加密)
- ✅ 低风险: 纯本地工具(数据不上传)

---

### 类型5: 声誉风险

**定义**: 产品失败或负面事件可能导致长期负面影响

**高风险场景**:
- 涉及社会公众利益
- 容易引发舆论关注
- 失败影响个人信用
- 负面影响难以消除

**评估问题**:
```
□ 失败会被媒体报道吗?
□ 会影响个人职业发展吗?
□ 会导致用户强烈不满吗?
□ 负面影响会持续多久?
□ 是否涉及敏感话题?
```

**风险等级判断**:
- **高风险**: 可能导致严重负面舆论
- **中风险**: 可能导致局部负面影响
- **低风险**: 小范围影响,可控
- **无风险**: 几乎不会有负面影响

**案例**:
- ❌ 高风险: 涉及政治、宗教、种族的内容平台
- ⚠️ 中风险: ToB产品故障影响客户业务
- ✅ 低风险: 个人效率工具

## 维度1评分计算

### 评分公式

```
基础分: 5分

风险扣分:
- 每个高风险: -2分
- 每个中风险: -1分
- 每个低风险: -0.5分

最终得分 = max(0, 基础分 - 扣分)
```

### 得分解读

```
5分: 无明显风险,可放心做
4分: 有低风险,可控
3分: 有中等风险,需要风控措施
2分: 有高风险,需要规避方案
1分: 有多个高风险,高度危险
0分: 致命风险,不应该做
```

### 及格标准

- **得分 < 2分**: 不建议做(一票否决)
- **得分 < 3分**: 必须有风险缓解方案才能继续

---

## 风险缓解方案

当风险得分 < 3分时,检查是否有缓解方案:

### 缓解方法清单

```
□ 可以通过保险转移风险?
  例: 购买产品责任险、E&O保险
  
□ 可以通过免责声明降低责任?
  例: 明确"仅供参考,非专业建议"
  注意: 保护力度有限,不能完全规避
  
□ 可以通过调整产品定位规避风险?
  例: 从"医疗诊断"改为"健康记录"
  
□ 可以通过技术手段降低风险?
  例: 多重备份、冗余设计、人工审核
  
□ 可以通过合作转移风险?
  例: 与有资质的大公司合作,借用其资质和保障
  
□ 可以通过限制使用场景降低风险?
  例: 明确"不适用于生命攸关场景"
```

### 缓解方案评估

```markdown
## 风险缓解方案

### 风险: [风险描述]

#### 缓解方案1: [方案名称]
- **方法**: __
- **成本**: ¥__
- **有效性**: [高/中/低]
- **残余风险**: [高/中/低]
- **可行性**: [高/中/低]

#### 缓解方案2: ...

### 综合评估
- **最佳方案**: __
- **缓解后风险等级**: [高/中/低/无]
- **是否可接受**: [是/否]
```

### 不可缓解的风险

以下风险通常**无法有效缓解**,应该放弃:

1. **生命安全风险**: 个人/小团队无法承担
2. **重大财产损失风险**: 即使有保险,赔偿也有限
3. **刑事法律风险**: 无法通过民事手段转移
4. **大规模隐私泄露**: 一旦发生,后果严重且不可逆

## 使用场景

### 场景1: 初步筛选

在三道防火墙(防火墙1)时使用,快速识别致命风险

### 场景2: 详细评估

在七维度评估时,深度分析各类风险的具体影响

### 场景3: 风险管理

产品上线后,持续监控和评估风险变化

## 常见误区

### 误区1: 免责声明万能论

❌ **错误**: "加个免责声明就没事了"
✅ **正确**: 免责声明在法律上保护力度有限,尤其涉及人身安全和重大财产损失

### 误区2: 小概率事件忽略论

❌ **错误**: "出问题的概率很低,不用担心"
✅ **正确**: 小概率事件总会发生,一旦发生个人无法承受

### 误区3: 参考大厂做法

❌ **错误**: "大厂都在做,我也可以"
✅ **正确**: 大厂有法务团队、保险、资金储备,个人没有

### 误区4: 技术能解决一切

❌ **错误**: "我会做好安全措施,不会出问题"
✅ **正确**: 没有绝对安全的系统,要考虑被攻击/被破解的可能

### 误区5: 事后补救心态

❌ **错误**: "出了问题再说"
✅ **正确**: 涉及风险的问题,事前预防远比事后补救重要

## 决策建议

### 立即放弃的情况

任何一项为"是",立即放弃:

```
□ 有人身安全高风险
□ 有重大财产损失高风险
□ 有严重法律风险且无合规路径
□ 维度1得分 < 2分
□ 无法有效缓解风险
```

### 需要调整的情况

满足以下条件可以考虑调整后继续:

```
□ 维度1得分 2-3分
□ 有明确的风险缓解方案
□ 缓解成本可接受
□ 缓解后风险等级降为中/低
□ 最坏情况可承受
```

### 可以继续的情况

```
□ 维度1得分 ≥ 3分
□ 最坏情况可承受
□ 已有基本风控措施
```
