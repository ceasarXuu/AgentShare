# JTBD 与 5-Why 根因分析

## Overview
从“用户说要什么”切换到“用户真正要解决什么”。

## Workflow
1. 收集典型用户原话并提炼表面需求
2. 写出JTBD陈述(情境+动机+期望结果)
3. 对关键痛点执行5-Why深挖
4. 合并出可落地的核心价值主张
5. 输出差异化设计方向

## JTBD陈述模板
- 当我遇到[情境]时
- 我想要[动机/需求]
- 这样我可以[期望结果]

## 5-Why实践
- 每次追问都记录事实依据
- 避免“因为我觉得”式解释

## Output
- JTBD陈述列表
- 5-Why根因链与核心需求
- 产品价值主张草案

## Guardrails
- JTBD必须对应真实场景与原话
- 根因要可被产品手段影响
