# Business Opportunity Assessment - 好生意评估

## 技能概述

这个技能实现了完整的"七维度好生意评估模型",专为个人开发者和小团队设计。它帮助快速判断一个产品方向是否值得投入时间和资源。

## 核心评估模型

```
好生意 = 合理的成本 × 可预期的收益 × 可承受的风险 × 可控的复杂度 × 可持续的运营
```

## 七个评估维度

### 优先级说明
- ⭐⭐⭐ (最重要,一票否决): 维度 1-3
- ⭐⭐ (重要): 维度 4-6
- ⭐ (加分项): 维度 7

### 评分标准
- 每个维度: 0-5 分
- 5分: 非常适合
- 4分: 适合
- 3分: 一般
- 2分: 需要注意
- 1分: 有明显问题
- 0分: 不可接受

### 及格标准
- 任何维度 < 2分: 不建议做
- 总分 < 21分 (平均3分): 不建议做
- 前三个维度必须 ≥ 3分

## 评估执行流程

### Step 1: 维度1 - 责任与风险 ⭐⭐⭐

**评估内容**:

| 风险类型 | 评估问题 | 风险等级 |
|---------|---------|---------|
| 人身安全 | 失败会导致人员伤亡吗? | 高/中/低/无 |
| 财产损失 | 会导致用户重大财产损失吗? | 高/中/低/无 |
| 法律责任 | 有明确的法律风险吗? | 高/中/低/无 |
| 隐私泄露 | 涉及敏感个人数据吗? | 高/中/低/无 |
| 声誉风险 | 失败会导致长期负面影响吗? | 高/中/低/无 |

**评分公式**:
```
风险得分 = 5 - (高风险数×2 + 中风险数×1 + 低风险数×0.5)
```

**风险缓解检查** (得分<3分时):
- 可以通过保险转移风险?
- 可以通过免责声明降低责任?
- 可以通过调整产品定位规避风险?
- 可以通过技术手段降低风险?
- 可以通过合作转移风险(找更大的主体)?

**输出**: 风险得分、风险清单、缓解建议

---

### Step 2: 维度2 - 成本结构 ⭐⭐⭐

**评估内容**:

**成本拆解**:
- 启动成本: 开发、设计、硬件、办公
- 固定成本(月): 服务器、第三方服务、人力、其他
- 变动成本(单用户): 存储、流量、客服

**关键指标**:
```
启动成本健康度 = 启动成本 / 可用资金
理想值: <0.5 (留一半做运营)

月度固定成本比率 = 月固定成本 / 预期月收入
理想值: <0.3 (固定成本不超过收入的30%)

盈亏平衡用户数 = 月固定成本 / (客单价 - 单用户变动成本)
理想值: <1000 (否则太难达到)

达到盈亏平衡时间
理想值: <12个月
```

**评分标准**:
- 启动成本 < 10万: +1分
- 月固定成本 < 1万: +1分
- 盈亏平衡用户数 < 500: +1分
- 变动成本占比 < 20%: +1分
- 12个月内能盈亏平衡: +1分

**输出**: 成本得分、成本明细表、盈亏平衡分析

---

### Step 3: 维度3 - 收益模型 ⭐⭐⭐

**评估内容**:

**收入来源评估**:
- 主要收入源: 收入占比、客单价、转化率、可行性
- 次要收入源: 同上
- 潜在收入源: 同上

**关键假设检验**:
识别最重要的3个假设,并明确验证方法

**保守收入预测**:
```
第1个月: 用户数、付费率、客单价、月收入
第3个月: 同上
第6个月: 同上
第12个月: 同上
```

**敏感性分析**:
- 如果用户数只达到预期的50%
- 如果付费率只有预期的50%
- 如果客单价降低30%
→ 是否仍能覆盖成本?

**评分标准**:
- 有明确的付费先例: +1分
- 客单价 > ¥50/月 或 > ¥200/年: +1分
- 预期付费率 > 10%: +1分
- 6个月内月收入 > 月成本: +1分
- 多个收入来源: +1分

**输出**: 收益得分、收入预测表、敏感性分析、关键假设清单

---

### Step 4: 维度4 - 技术复杂度 ⭐⭐

**评估内容**:

**技术模块评估**:
- 前端开发: 难度(高/中/低)、是否有先例、预估工时
- 后端开发: 同上
- 数据库设计: 同上
- 第三方集成: 同上
- 特殊功能: 同上

**技术风险识别**:
- 需要重大技术突破? (如AI算法创新)
- 依赖未成熟的技术栈?
- 需要大量硬件/系统级开发?
- 有严格的性能/可靠性要求?
- 需要处理复杂的边界情况?
- 依赖外部平台的不稳定接口?

**评分公式**:
```
基础分: 3分
技术风险调整:
- 无技术风险项: +2分
- 有1-2个技术风险: +1分
- 有3-4个技术风险: 0分
- 有5个以上技术风险: -1分

开发工时调整:
- <500小时: +2分
- 500-1000小时: +1分
- 1000-2000小时: 0分
- >2000小时: -1分
```

**输出**: 技术得分、技术风险清单、工时估算

---

### Step 5: 维度5 - 运营负担 ⭐⭐

**评估内容**:

**日常运营需求**:
- 内容审核: 频率、人力需求、可否自动化
- 客服支持: 24/7/工作时间/异步、人力需求、可否自动化
- 系统监控: 24/7/定期检查、人力需求、可否自动化
- 数据分析: 频率、人力需求、可否自动化
- 内容更新: 频率、人力需求、可否自动化

**运营复杂度检查**:
- 需要24/7待命?
- 故障必须立即响应?
- 需要人工审核内容?
- 需要频繁处理纠纷?
- 需要维护复杂的用户关系?
- 需要持续产出内容?

**生活质量影响**:
- 每周工作时间: ___小时
- 能否正常休假: 能/不能
- 是否影响睡眠: 是/否
- 心理压力程度: 高/中/低
- 是否可以外包: 完全/部分/不可以

**评分标准**:
- 无需24/7待命: +1分
- 客服可以异步处理: +1分
- 大部分任务可自动化: +1分
- 每周运营<20小时: +1分
- 不影响正常生活: +1分

**输出**: 运营得分、运营需求表、生活质量影响分析

---

### Step 6: 维度6 - 市场与竞争 ⭐⭐

**评估内容**:

**市场规模**:
```
TAM (总体市场): ___________
SAM (可服务市场): ________
SOM (可获得市场,第一年): ________
```

**市场成熟度**:
- 成熟市场 (竞争激烈但需求明确)
- 成长市场 (有竞品但还有空间)
- 新兴市场 (需要教育但潜力大)
- 蓝海市场 (几乎没有竞品)

**竞品分析** (至少3个):
- 优势、劣势、定价、市场份额

**差异化优势**:
- 核心差异点 (至少3个)
- 用户是否真正在意这些差异?
- 这些差异是否容易被模仿?

**获客成本评估**:
```
CAC (获客成本): ¥______
LTV (用户生命周期价值): ¥______
LTV/CAC 比率: ______
理想比率: >3 (LTV是CAC的3倍以上)
```

**评分标准**:
- SOM > 10万用户: +1分
- 有明确的差异化优势: +1分
- LTV/CAC > 3: +1分
- 竞品有明显短板可切入: +1分
- 市场增长中且有需求验证: +1分

**输出**: 市场得分、竞品分析表、差异化优势分析、获客策略

---

### Step 7: 维度7 - 可扩展性 ⭐

**评估内容**:

**用户规模扩展**:
```
从1000到10000用户:
- 技术架构是否需要重构: 是/否
- 成本是否线性增长: 是/否
- 是否需要更多人力: 是/否

从10000到100000用户:
- 技术架构是否需要重构: 是/否
- 成本是否线性增长: 是/否
- 是否需要更多人力: 是/否
```

**产品功能扩展**:
- 可以添加新功能而不影响核心功能
- 可以扩展到相关品类
- 可以开发企业版
- 可以发展平台/生态

**商业模式扩展**:
- 可以增加新的收入来源
- 可以提高客单价
- 可以从B2C扩展到B2B
- 可以做品牌授权/加盟

**评分标准**:
- 用户规模扩展成本低: +2分
- 有明确的功能扩展路径: +1分
- 有商业模式升级可能: +1分
- 可以发展平台效应: +1分

**输出**: 扩展性得分、扩展路径分析

---

## 决策树逻辑

执行完七个维度评估后,按以下决策树判断:

```
1. 任何维度得分 < 2分?
   → 是: ❌ 不建议做
   → 否: 继续

2. 前三个维度(风险、成本、收益)都 ≥ 3分?
   → 否: ⚠️ 需要解决核心问题
   → 是: 继续

3. 总分 < 21分?
   → 是: ⚠️ 整体评分偏低,需要重大调整
   → 否: 继续

4. 总分 ≥ 28分?
   → 是: ⭐ 优秀的商业机会!
   → 否: ✅ 可以做,但需要持续优化
```

## 使用示例

```
作为 NBResearcher,当用户要求评估一个产品 idea 时:

1. 确认已通过"三道防火墙"检查
2. 逐个维度收集信息并评分
3. 使用本技能的评分标准计算各维度得分
4. 应用决策树逻辑
5. 输出标准化评估报告
6. 提供明确的"做/不做"建议
```

## 注意事项

1. **保守原则**: 收益保守估计,成本悲观估计
2. **风险优先**: 先看最坏情况,后看收益
3. **三角验证**: 每个关键假设都需要多个数据源支撑
4. **诚实评估**: 不要为了"想做"而降低标准
5. **生活优先**: 如果会严重影响生活质量,即使评分高也要慎重
