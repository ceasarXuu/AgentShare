# 好生意快速评估工作流

## 概述

这是一个 30 分钟的快速评估工作流,帮助用户快速判断一个产品 idea 是否值得投入更多时间深入研究。

**用途**: 快速筛选多个 idea,找出最值得深入研究的方向
**时间**: 30 分钟
**输出**: 初步评估报告 + 做/不做建议

## 执行步骤

### Step 1: 收集基本信息 (5分钟)

请用户提供以下信息:

```markdown
1. **产品名称/描述**: 一句话描述这个产品是什么
2. **目标用户**: 谁会用这个产品?
3. **核心痛点**: 解决什么问题?
4. **预估启动成本**: 大概需要多少钱启动?
5. **预估开发时间**: 做出 MVP 需要多久?
```

### Step 2: 三道防火墙检查 (10分钟)

使用 `agents/kuko/SKILLS/three-firewall-check/SKILL.md` 执行检查:

#### 防火墙1: 致命风险检查

```markdown
□ 会导致人身伤害或生命危险?
□ 会导致重大财产损失(用户的)?
□ 失败会有严重法律后果?
□ 需要承诺可靠性但无法保证?
□ 涉及敏感群体的安全责任?
```

**任何一项为"是" → 终止评估,建议放弃**

#### 防火墙2: 基本可行性检查

```markdown
□ 能在6个月内用<50万启动?
□ 能在3个月内做出可测试的MVP?
□ 有明确的首批目标用户?
□ 技术实现路径清晰?
□ 前3个月的行动计划清晰?
```

**超过2个"否" → 终止评估,建议调整范围**

#### 防火墙3: 个人适配性检查

```markdown
□ 真的想做这件事(不是为了赚快钱)?
□ 有相关领域经验或强烈兴趣?
□ 能接受前6个月几乎没有收入?
□ 能承受失败(时间和资金损失)?
□ 家人支持?
```

**少于4个"是" → 给出警告,建议等待时机**

### Step 3: 快速八项评估 (10分钟)

| 评估项 | 问题 | 理想答案 | 实际情况 | 得分(1-5) |
|--------|------|---------|---------|-----------|
| **风险** | 最坏情况是什么? | 损失时间和小额资金 | | |
| **启动成本** | 需要多少钱启动? | <10万 | | |
| **MVP时间** | 多久能做出可测试版本? | <3个月 | | |
| **首批用户** | 能否找到100个种子用户? | 明确知道在哪 | | |
| **收费意愿** | 用户愿意为此付费吗? | 有付费先例 | | |
| **运营负担** | 需要24/7待命吗? | 否 | | |
| **竞争状况** | 市场上有类似产品吗? | 有但不够好 | | |
| **技术门槛** | 技术上能实现吗? | 明确可行 | | |

**评分标准**:
- 5分: 完美符合理想答案
- 4分: 基本符合
- 3分: 一般
- 2分: 有问题
- 1分: 严重问题

### Step 4: 生成初步报告 (5分钟)

根据评估结果生成报告:

```markdown
# 好生意快速评估报告

## 项目: [项目名称]
## 评估日期: [日期]

### 三道防火墙结果

| 防火墙 | 结果 |
|--------|------|
| 致命风险检查 | ✅通过 / ❌不通过 |
| 基本可行性检查 | ✅通过 / ⚠️需调整 / ❌不通过 |
| 个人适配性检查 | ✅通过 / ⚠️需注意 / ❌不建议 |

### 快速八项评估

| 评估项 | 得分 |
|--------|------|
| 风险 | __/5 |
| 启动成本 | __/5 |
| MVP时间 | __/5 |
| 首批用户 | __/5 |
| 收费意愿 | __/5 |
| 运营负担 | __/5 |
| 竞争状况 | __/5 |
| 技术门槛 | __/5 |
| **总分** | **__/40** |

### 初步结论

[基于评估给出明确建议]

- 总分≥28: ⭐ **值得深入研究** - 建议进行详细七维度评估
- 总分20-27: ⚠️ **需要优化** - 建议调整后重新评估
- 总分<20: ❌ **不建议做** - 建议放弃或大幅调整方向

### 核心问题点

1. ...
2. ...
3. ...

### 建议的下一步

- [ ] ...
- [ ] ...
```

## 决策指南

### 值得深入研究 (继续)

满足以下条件:
- 三道防火墙全部通过
- 快速评估总分 ≥ 28
- 没有任何单项得分 < 2

**下一步**: 使用 `/detailed-evaluation` 进行详细七维度评估

### 需要优化 (调整后重评)

满足以下条件:
- 三道防火墙有"需调整"项
- 快速评估总分 20-27
- 有单项得分 < 2 但可优化

**下一步**: 针对问题项调整方案,重新评估

### 不建议做 (放弃)

满足以下条件:
- 三道防火墙任一项不通过
- 快速评估总分 < 20
- 核心问题无法解决

**下一步**: 放弃该方向,或大幅调整产品定位

## 使用示例

```
用户: 我想做一个帮助老人紧急呼救的App,叫"无恙"

Agent: 好的,让我们用30分钟快速评估这个idea...

[执行三道防火墙检查]

防火墙1检查结果:
- 会导致人身伤害或生命危险? ✓ 是 (如果通知失败)
- 涉及敏感群体的安全责任? ✓ 是 (老人安全)

❌ 致命风险检查不通过

结论: 这个产品方向存在致命风险,不建议个人开发者/小团队做。
建议: 考虑调整为"老人关怀提醒"App,记录关怀历史,不承诺紧急救助功能。
```

## 注意事项

1. **保持客观**: 不要因为"想做"而降低标准
2. **风险优先**: 有疑虑就不做
3. **快速决策**: 30分钟内完成,不要过度分析
4. **诚实评估**: 每个答案都要有依据
