# 七维度详细评估工作流

## 概述

这是一个 2-3 小时的详细评估工作流,对通过快速评估的产品 idea 进行深度分析,使用七维度评估模型,输出完整的决策报告。

**前置条件**: 已通过 `/good-business-assessment` 快速评估
**时间**: 2-3 小时
**输出**: 详细评估报告 + 明确的做/不做决策 + 行动计划

## 执行步骤

### Step 1: 确认前置条件 (5分钟)

验证:
- 三道防火墙已全部通过
- 快速评估总分 ≥ 20
- 用户已准备好投入2-3小时进行详细评估

### Step 2: 维度1 - 责任与风险评估 (20分钟)

使用 `agents/kuko/SKILLS/responsibility-risk-assessment/SKILL.md`

**评估五类风险**:
1. 人身安全风险
2. 财产损失风险
3. 法律责任风险
4. 隐私泄露风险
5. 声誉风险

**评分**: 5 - (高风险×2 + 中风险×1 + 低风险×0.5)

**输出**: 风险清单、最坏情况分析、缓解方案(如需要)

### Step 3: 维度2 - 成本结构分析 (25分钟)

使用 `agents/kuko/SKILLS/cost-structure-analysis/SKILL.md`

**分析内容**:
- 启动成本拆解
- 月固定成本
- 单用户变动成本
- 盈亏平衡点计算
- 敏感性分析

**评分标准**:
- 启动成本<10万: +1分
- 月固定成本<1万: +1分
- 盈亏平衡用户数<500: +1分
- 变动成本占比<20%: +1分
- 12个月内能盈亏平衡: +1分

### Step 4: 维度3 - 收益模型验证 (25分钟)

使用 `agents/kuko/SKILLS/revenue-model-validation/SKILL.md`

**分析内容**:
- 收入来源评估
- 关键假设识别
- 保守收入预测(1/3/6/12个月)
- 敏感性分析
- 付费先例研究

**评分标准**:
- 有明确的付费先例: +1分
- 客单价>¥50/月或>¥200/年: +1分
- 预期付费率>10%: +1分
- 6个月内月收入>月成本: +1分
- 多个收入来源: +1分

### Step 5: 维度4 - 技术复杂度评估 (15分钟)

使用 `agents/kuko/SKILLS/business-opportunity-assessment/SKILL.md` 中的技术评估部分

**评估内容**:
- 技术模块难度评估
- 技术风险识别
- 开发工时估算

**评分**:
- 基础分3分
- 无技术风险+2分 / 1-2个风险+1分 / 3-4个风险0分 / 5+风险-1分
- 工时调整: <500小时+2分 / 500-1000小时+1分 / 1000-2000小时0分 / >2000小时-1分

### Step 6: 维度5 - 运营负担分析 (20分钟)

使用 `agents/kuko/SKILLS/operation-burden-analysis/SKILL.md`

**分析内容**:
- 日常运营任务清单
- 运营复杂度检查
- 生活质量影响评估
- 自动化可行性分析

**评分标准**:
- 无需24/7待命: +1分
- 客服可以异步处理: +1分
- 大部分任务可自动化: +1分
- 每周运营<20小时: +1分
- 不影响正常生活: +1分

### Step 7: 维度6 - 市场与竞争分析 (25分钟)

使用现有 Skills:
- `agents/kuko/SKILLS/market-sizing/SKILL.md`
- `agents/kuko/SKILLS/competitive-gap-analysis/SKILL.md`

**分析内容**:
- 市场规模(TAM/SAM/SOM)
- 市场成熟度
- 竞品分析(至少3个)
- 差异化优势
- 获客成本评估(CAC/LTV)

**评分标准**:
- SOM>10万用户: +1分
- 有明确的差异化优势: +1分
- LTV/CAC>3: +1分
- 竞品有明显短板可切入: +1分
- 市场增长中且有需求验证: +1分

### Step 8: 维度7 - 可扩展性评估 (10分钟)

**评估内容**:
- 用户规模扩展(1K→10K→100K)
- 产品功能扩展
- 商业模式扩展

**评分标准**:
- 用户规模扩展成本低: +2分
- 有明确的功能扩展路径: +1分
- 有商业模式升级可能: +1分
- 可以发展平台效应: +1分

### Step 9: 综合决策 (15分钟)

#### 计算总分

| 维度 | 得分 | 权重 |
|------|------|------|
| 1. 责任与风险 | __/5 | ⭐⭐⭐ |
| 2. 成本结构 | __/5 | ⭐⭐⭐ |
| 3. 收益模型 | __/5 | ⭐⭐⭐ |
| 4. 技术复杂度 | __/5 | ⭐⭐ |
| 5. 运营负担 | __/5 | ⭐⭐ |
| 6. 市场与竞争 | __/5 | ⭐⭐ |
| 7. 可扩展性 | __/5 | ⭐ |
| **总分** | **__/35** | |

#### 应用决策树

```
1. 任何维度得分 < 2分?
   → 是: ❌ 不建议做
   → 否: 继续

2. 前三个维度(风险、成本、收益)都 ≥ 3分?
   → 否: ⚠️ 需要解决核心问题
   → 是: 继续

3. 总分 < 21分?
   → 是: ⚠️ 整体评分偏低,需要重大调整
   → 否: 继续

4. 总分 ≥ 28分?
   → 是: ⭐ 优秀的商业机会!
   → 否: ✅ 可以做,但需要持续优化
```

## 质量检查清单

完成评估前确认:

```
□ 七个维度都已完成评估
□ 每个维度都有具体得分和依据
□ 前三个维度(风险、成本、收益)分析充分
□ 已识别所有关键风险并有缓解方案
□ 成本估算包含50%余量
□ 收益预测采用保守估计
□ 决策建议明确(做/不做)
□ 行动计划具体可执行
□ 止损线明确定义
```

## 注意事项

1. **不要跳过维度**: 每个维度都必须评估
2. **保持保守**: 宁可错过,不要做错
3. **关注前三维度**: 风险、成本、收益是核心
4. **诚实评估**: 有疑虑就标注出来
5. **明确决策**: 不要给模糊的建议
