# 信源筛选与可信度评估

## Overview
用信源分级与筛选清单降低噪声，确保结论来自高价值、可交叉验证的来源。

## Workflow
1. 列出候选信源并标注来源类型
2. 按可信度金字塔分级并标记优先级
3. 用4问清单逐条筛选并记录结果
4. 标记需交叉验证的条目与缺口
5. 输出可用信源清单与待补充信源

## 信源可信度金字塔
- Level 1 用户原始行为: Reddit/X.com/App Store差评/Apple Community
- Level 2 一手数据平台: Sensor Tower/App Annie(App data.ai)/Appfigures/Google Trends/Apple Search Ads
- Level 3 行业洞察: WWDC/垂直专家博客/咨询报告
- 避免使用: PR新闻、媒体排行榜、无验证榜单

## 4问筛选清单
- 检查动机: 是否有明显PR或流量驱动
- 验证样本量: 是否足够大且能形成共识
- 确认时效性: 是否在12个月内或标注过时
- 判断可交叉验证性: 是否能与其他来源互证

## Output
- 信源清单: 名称/级别/动机/样本量/时效性/可交叉验证性
- 高风险信源标记与需补充的替代信源列表

## Guardrails
- 无三角验证不下关键结论
- 过时数据必须标注并谨慎使用
- 优先用户原始行为数据而非观点性内容
