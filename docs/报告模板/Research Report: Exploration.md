# 模板一：探索研究型报告（Exploratory Research Report）

**适用场景**
- 没有指定题目 Agent 自主扫描后，选择一个“最值得关注的机会”进行深度论证

**核心目标**
- 说明“为什么这个方向值得你继续投入注意力”

# 《商业机会探索研究报告》

## 0. Executive Overview（执行概览）

### 研究结论（一句话）
- [在此填写一句话结论]

**示例**
- 本次探索聚焦于【用户角色 × 场景 × 核心行为】，这是一个当前被低估、但具备明确工具化与商业化潜力的机会。

### 为什么是现在（事实驱动）
- 【行为 / 技术 / 平台变化】导致用户在该场景下的操作频率或成本显著上升
- 现有解决方案在【核心动作】上存在明显结构性不足
- 用户已形成稳定的高成本替代行为或付费行为

### 机会类型判定
- [ ] 系统能力不足型
- [ ] 过度工具化型
- [ ] 定价 / 价值失衡型
- [ ] 行为外包型

## 1. 机会的产生背景（Opportunity Genesis）

### 1.1 探索触发点（Exploration Trigger）

**说明本次机会是如何被发现的**
- 来源于哪一类扫描或信号
- 搜索行为异常
- 用户讨论集中
- 定价区间异常
- 平台能力变化

**目标**
- 证明这是一个被数据“推出来”的机会，而非主观设想

### 1.2 为什么选择这一方向深入研究

从以下三个角度进行正向论证：
- 该方向是否具备足够聚焦的用户与行为边界
- 是否存在清晰的「动作 → 结果」闭环
- 是否天然适合通过单点工具而非平台化解决

## 2. 用户是谁，以及问题发生在什么情境下

### 2.1 核心用户画像（Contextual Persona）

仅回答三件事：
- 用户角色（职业 / 使用身份）
- 典型触发场景
- 用户试图完成的具体目标

### 2.2 用户真实行为证据

**本节禁止总结性语言，只呈现证据**

允许内容：
- 用户原始讨论摘录（可翻译）
- 行为描述（例如：手动整理、Excel、截图、重复导出）

## 3. 当前解决方案的结构性特征

### 3.1 市场现有方案的共性结构

分析维度：
- 功能组织方式
- 使用路径复杂度
- 定价模型

**注意**
- 不进行情绪评价，不点名批评单一产品

### 3.2 由结构自然暴露的机会空间

回答一个问题：
- 如果只围绕「核心动作」重新设计工具，会出现什么新的价值空间？

## 4. 商业化逻辑分析（正向）

### 4.1 用户为何愿意为此付费

基于以下至少两类证据进行论证：
- 已存在的直接付费行为
- 已付出的高时间 / 精力成本
- 已承担的风险成本（错误、合规、遗漏）

### 4.2 可行的商业模式路径

列举 2–3 种，并说明：
- 为什么与该行为天然匹配
- 为什么不会破坏使用体验

### 4.3 价格锚点的现实基础

价格判断必须来自：
- 市场已有定价区间
- 用户讨论中的心理价位
- 替代方案的隐性成本

## 5. 为什么这是一个“小而美”的工具机会

从以下三点进行论证：
- 功能为何可以高度收敛
- 为什么不需要平台 / 社区 / 内容生产
- 为什么长期维护成本可控

## 6. 总结：为何值得进入下一阶段

**本节不讨论成功或失败，只讨论是否值得继续投入研究或验证**

可能的结论形式：
- 值得进入 MVP 验证阶段
- 值得立项进行定向研究
- 值得作为中期机会持续观察
